## 0. 一句話 + 給 開發者 的任務

把獵人角做成一座你走得進去、改變得了的城。志工進來會驚「原來這裡這麼多事可以做」，選一個身分、一關一關闖，每完成一處，那一塊就在**真地圖上點亮**；三福全亮 → 看見整個獵人角活過來 → 被真實邀請來付出兩小時。

**開發者 要做的：** 以 `index.html` 為底，按本規格擴充成「三福分區 hub ＋ 多關（各關不同玩法、各帶真故事真照片）＋『看看現在的獵人角』景深街景 ＋ 真實邀請」的完整單頁遊戲，部署到 GitHub Pages。

**核心不是關卡數，是「依存性」（見 §3.5）**：玩家自己選、自己養一株小芽、自己墾出一座會累積記憶的獵人角——像電子雞、像自己開墾的田地，會想一直回來。**絕不要交一個高中生週末就寫完的淺成品。**

## 1. 鐵軌（不可違反 · 違反即停）

- 離線可玩 · 純 Canvas 渲染（不用 3D 引擎）· 部署 GitHub Pages。
- **畫面零演算法**（這不是 JOK；演算法只活在 JOK，那邊保留）。
- **AI 不進孩子畫面**（只能在後台；不即時把未審內容秀給未成年）。
- 志工穿**藍天白雲**制服（藍上白下，是獵人角的「通行證」）；顏色像、但**不放官方 Logo**。
- 場景一律**白天**（入夜該區較危險；白天辨識度高、制服即通行證）。
- 真實照片**馬賽克規則（明暺定 · 嚴格遵守）**：**所有孩子的臉一律打馬賽克**（即使孩子很想被看見、覺得領獎很光榮，為安全一律打）；**醫師也打**（未取得同意，省去詢問）；志工、校長、老師可不打。**不確定 → 打。** JOK 照片幾乎都已是馬賽克版可直接用；若遇到沒打的，先補打再用。
- **社區是英雄、不是可憐人**：「點亮」＝社區點亮自己，玩家是被邀進來的人，不是來拯救。
- **受災民眾不進遊戲當角色**（同意被記錄 ≠ 同意成為遊戲畫面）。
- 場景**不放柱子**。
- 三語 EN／ZH／ES，全程可切換。
- 明暺全部篩選；任何真實人物內容先過她。

## 1.1 可讀性 / 無障礙（明暺實測 · 必守）

地圖很美但「很花」；節點若用半透明毛玻璃＋淺色圖案，老花的人看過去就是一團彩色，找不到關卡、不知如何點。開發者 必須：

- 節點 = 一眼看得出是「可點的 target」：**實心底、深色粗環、深色外暈**把它從地圖抬起來；**不要半透明毛玻璃**。
- 名稱／星級／狀態放進**實心深色膠囊**，任何地圖底都讀得清（不要只靠白字陰影）。
- 可玩的關最醒目（加大、金光、脈動）；即將的仍是**清楚的實心節點**，不要半隱形。
- 節點彼此**拉開不重疊**；地圖底色再壓深一階，讓節點浮出。
- 為老花設計：字夠大、對比夠強。腳手架 `index.html` 已示範這版節點樣式，照它或做得更清楚。
- 可選（從根本去花）：用**三福分區收納**節點——點一個福 → 才展開該福的活動，hub 一眼只看三個大目標。

## 3. 三福骨架（敘事主軸 · 從這裡寫起）

整個遊戲圍繞淑雲師姐的**獵人角三福**組織。開場（推門後）用一段三福的話起頭，三福各是地圖上的一個分區，分區內的活動完成就點亮那一福；三福全亮 → 整個獵人角亮。

| 福 | 含義 | 收哪些活動（提案，明暺校正） |
|---|---|---|
| **幸福校園** | 獵人角當地、Bret Harte 小學的孩子深耕（≠ 人文學校，見 §2） | 課後輔導、**全勤獎**、素食烹飪班 |
| **幸福家園** | 支持每個家、每個人的溫飽與安身 | 食物發放、冬季毛毯、RV Park 關懷 |
| **幸福社區** | 把整個社區照顧起來 | 義診、環保小尖兵、遊民熱食 |

> Hub 上以三個暖色分區（或三個聚落）呈現三福；節點落在各自分區。完成一福內全部活動 → 該福亮起一個總標記（例如一盞大燈／一棵小樹）。

## 3.5 核心循環 · 依存性（「好不好玩」的命脈 · 必寫）

明暺要的不是一串各自獨立的小遊戲，而是一座**你自己選、自己養、自己墾出來的獵人角**——像電子雞、像自己開墾的一方田地，養出來的東西捨不得丟，會想一直回來。沒有這一層，就是不及格。

**1. 自己選（agency · 非線性）** — 玩家自由決定今天進哪個福、哪個關、扮哪個角色。沒有強制順序；hub 就是「你決定今天把心力放哪」的地圖。選角色不只換稱呼，會影響你能接哪些關、社區怎麼回應你。

**2. 自己養一株小芽（電子雞般的依存性）** — 玩家有一株屬於自己的**小芽**（呼應遊戲名與「2006 一把生米 → 善的種子」）。每做一件好事，小芽長一點；愈耕耘愈茁壯，最後長成一棵樹。小芽就是你的電子雞——你親手養大的，捨不得放著不管。**跨次保存**：下次打開，牠還在、又長了一截。

**3. 自己墾一方田地（永久累積的世界）** — 每一次付出都**永久改變地圖、而且累積**：清乾淨的街角會一直乾淨；訪視過的家窗口亮起暖燈；種下的樹下次來長更高。獵人角是你一鋤一鋤墾出來的，它記得你做過什麼；空著會想回來繼續墾。

**4. 手作感（這是「我」做的）** — 可命名自己的志工、選角色與外觀、佈置自己負責的那個角落。人對親手做的東西有感情，才會想回來顧。

**5. 回來的理由（開放式，不收尾）** — 社區持續活著：小芽持續長、抓 FB／IG 當天有新的真實需要冒出來、墾過的地可以再深耕。沒有「破關結束」，只有「今天我又來了」。這正是現在最紅那批 cozy 經營／養成遊戲讓人上癮的點，用在「善」上。

**6. 技術：持久化（依存性的關鍵）** — 部署在 GitHub Pages（非 線上預覽工具），用 **localStorage** 存：小芽成長值、已點亮／已開墾的地圖狀態、玩家自訂（名字、角色、佈置）。下次打開接著長。（腳手架在 線上預覽裡 localStorage 可能被擋，用記憶體 fallback；真正部署到 Pages 用 localStorage 才有「養出來」的依存性。）

## 4. 遊戲流程

```
推門（0529 開場殼，一個字不動）
 → 選身分（義診醫護／課輔老師／環保／食物／香積／哪裡需要去哪裡）
 → 獵人角 hub：真實地圖底 + 三福分區 + 活動節點（各帶星級 + 真故事預覽）
        每個節點點下去 = 看它的「來龍去脈 + 星級 + 玩法」→ 進關
 → 關卡：各關不同玩法（見 §5）→ 完成 → 那一塊在真地圖上點亮、社區活力上升、回 hub
 → 三福逐步點亮
 → 「看看現在的獵人角」=【整張地圖 + Marble 景深街景全景】（見 §8）
 → 真實邀請（AG 食物發放）+ FB / IG
```

身分（role）影響：進場高亮對應分區的活動；之後可擴充成「以這個身分帶動一個 project」。完成的關卡持久點亮、跨關累積（這一關蓋的下一關接著亮）。

### 5.1 各關真實玩法細節（明暺校正 · 開發者 照這個寫）

**全勤獎（幸福校園 · ★ · 慶祝・抽獎）**
真實獎勵機制：
- **Tiger Paw 點數**：每個孩子依當週良好表現得點數，週五午餐時間到「**雜貨店**」兌換心儀禮品；點數在**講臺**統一發放。
- **現打新鮮水果 Smoothie**：全勤的額外獎，午餐時間現場打給全勤的孩子喝。
玩法：像 JOK 全勤獎那關——很多獎品可以抽（Tiger Paw 等）。做成「為每個全勤孩子鼓掌 → 抽獎／兌換」的慶祝節奏，溫暖、不淘汰、人人有得。

**幸福家園 訪視關懷（家園 · 參考 JOK 第一關 Mrs. Garcia）**
我們做定期的**訪視與關懷**。遊戲裡只呈現「上門看一個家、陪一段、送上需要的」那份溫暖，玩法參考 JOK L1 Mrs. Garcia。
鐵則：**絕不寫負面內容**（家暴、安置等一律不出現），只提訪視關懷；受關懷的家庭不指認、不獵奇。

**RV Park 關懷（家園 · ★★★ · 配對・備關懷包）**
我們幫 RV 夥伴準備盥洗與民生必需品。玩法 = 依需要配出一份關懷包，四類：
1. 牙刷、牙膏、漱口水
2. 洗髮精、沐浴乳
3. 拍淨／排淨之類
4. 毛巾或襪子
安靜、尊重、不打擾、不評斷。

**幸福社區 本土志工（社區 · Maya）**
幸福社區的核心 = **培養本土志工，讓社區自己照顧自己**。
Maya（化名）= 這裡第一顆「**黑珍珠**」受證委員，如今深耕社區、是社區窗口，負責所有聯絡事項。做成「在地的人站出來、成為這片土地的窗口與依靠」的故事（對應 JOK 那關 Maya）。

## 6. 資料結構（data-driven，孩子之後能直接加角色／關卡）

全部從資料驅動，新增關卡或角色只改資料、不動引擎。腳手架已有 `ACTIVITIES` / `ROLES` 雛形，擴成：

```js
const FU = ['campus','home','community'];           // 三福
const ACTIVITIES = [{
  id:'foodshare',
  fu:'home',                    // 屬於哪一福
  state:'play',                 // play | soon
  stars:1,                      // 1–3 難度
  mech:'serve-listen',          // 玩法種類（決定載入哪個 mini-engine）
  pos:{x:0.52,y:0.62},          // 在真地圖上的位置（分數）
  name:{zh,en,es},
  story:{zh,en,es},             // 來龍去脈
  jingsi:{zh,en,es},            // 靜心之門靜思語（過場）
  assets:{ photo:'clean_Mrs__Garcia.png', music:'foodshare.mp3' },
  neighbors:[ /* 各關自己的真人角色：名字、需求、心的一句話、膚色/衣色多元 */ ],
}, /* ... */];
const ROLES = [ /* medic, tutor, eco, food, kitchen, any，各帶三語 */ ];
const CHARACTERS = []; // 給 Summer Camp 小朋友共創，往這裡加（characters.json）
```

`mech` 對應一個小引擎模組（serve-listen / match-give / route-deliver / observe-care / sort-clean / cook-combo / explore-care / puzzle / ceremony）。每個 mech 是一個可重用函式，吃 activity 資料、回傳一個可玩關。

**持久化 / 小芽存檔（§3.5 依存性的核心）** — 另存一份玩家狀態，部署到 Pages 用 localStorage 保存、開場載入：

